## Context

当前系统提供了一个 `timeseries_forecast` 工具用于预测时间序列数据，但缺乏专门的异常检测机制。为了提高业务指标异常检测的准确性和可用性，我们需要一个异常评估模块。该模块将使用预测结果作为基准来评估实际数据点，根据多个**正交因子**（偏离显著性、方向、业务体量、历史噪声频率、持续时间）分配 0 到 1 的异常分数。

## Goals / Non-Goals

**Goals:**
- 实现一个异常评估模块，对异常进行 0 到 1 的打分。
- 提供一个新的工具 `anomaly_detection`，集成预测和评估功能。
- 采用**相互正交**的多因素打分，避免对同一证据重复计分：偏离显著性、偏离方向、业务体量调制、历史噪声频率、异常持续时间。
- **代码规范**: 在后续代码实现中，所有代码注释必须保留为英文 (Keep comments in English during implementation)。

**Non-Goals:**
- 替换现有的 `timeseries_forecast` 工具。
- 在本次迭代中实现复杂的监督机器学习模型进行打分（我们将从基于规则/策略的打分机制开始）。
- 跨窗口的长期"异常持续时间"统计：本期持续时间仅在评估窗口内统计（窗口外的历史持续时长不计入）。

## Decisions

- **新工具 vs 修改现有工具**: 我们将创建一个新工具 `anomaly_detection.py` (位于 `tool` 目录下，仅做薄封装)，其核心算法逻辑放置于 `analysis/anomaly_detection.py`。`tool` 层不含业务逻辑，直接调用 `analysis` 层的 `evaluate_anomalies` 与 `format_anomaly_report`。`call_timeseries_service` 作为公开方法供 `analysis` 层程序化调用（返回结构化数据），与面向 Agent 返回格式化字符串的 `timeseries_forecast` 工具区分。
- **正交打分策略**: 每个点的分数由以下正交因子组合，最终窗口分数归一化到 [0, 1]：
  - **偏离显著性 (Deviation Significance)**: 以稳健噪声尺度 σ 衡量实际值偏离预测值的统计显著程度（z 分数）。在 3σ 处达到 0.5、6σ 处饱和为 1.0。这是唯一的"有多异常"信号，**取代了原先高度冗余的重构误差 + 边界违规两项**。
  - **偏离方向 (Direction)**: 区分下降/上涨，并支持对下降施加更高权重（`drop_weight`/`rise_weight`，默认中性 1.0），因为多数业务指标的"掉量"比"涨量"更严重。
  - **业务体量调制 (Volume Modulation)**: 由**预测的期望水平**（而非异常的实际值）驱动的 [0.6, 1.0] 乘子；高流量时刻保持满权重，低流量时刻被压低。基于期望水平可确保"掉零"这类严重异常不会被错误降分。
  - **历史噪声频率 (Historical Anomaly Frequency)**: 历史上经常越界（抖动）的噪声指标会被阻尼，降低误报。
  - **持续时间 (Duration)**: 窗口末尾连续异常点会提升分数。
- **稳健噪声尺度**: σ 基于一阶差分的 **MAD（中位数绝对偏差）** 估计，对趋势/周期不敏感（差分去除趋势）且对历史尖峰稳健（MAD 不被离群点抬高），仅在 MAD 退化时回退到标准差。
- **上下界来源**: 预测服务当前仅返回点预测（无置信区间），故 σ 来自历史序列的稳健估计；实现中已预留：若服务未来返回逐步不确定度 (`prediction_std`)，将优先使用。
- **评估窗口加权**: 采用**线性权重**（越新权重越大），比指数权重更温和，避免多步预测中误差最大的远端点（恰好权重最高）主导打分。

## Risks / Trade-offs

- **风险**: 基于规则的打分可能需要针对不同指标调整权重。
  - **缓解措施**: 暴露 `drop_weight`/`rise_weight` 等可调参数，并使用适用于典型流量/业务指标的合理默认值。
- **风险**: 多步预测的精度随预测步长衰减。
  - **缓解措施**: 采用线性窗口权重缓解远端误差主导；并预留使用逐步不确定度的接口。
- **风险**: 底层预测服务可能会失败或超时。
  - **缓解措施**: 优雅地处理预测错误，并向 Agent 返回信息丰富的错误消息。
