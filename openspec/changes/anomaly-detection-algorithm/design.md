## Context

当前系统提供了一个 `timeseries_forecast` 工具用于预测时间序列数据，但缺乏专门的异常检测机制。为了提高业务指标异常检测的准确性和可用性，我们需要一个异常评估模块。该模块将使用预测结果作为基准来评估实际数据点，根据多个因素（如重构误差、历史上下文、持续时间）分配 0 到 1 的异常分数。

## Goals / Non-Goals

**Goals:**
- 实现一个异常评估模块，对异常进行 0 到 1 的打分。
- 提供一个新的工具 `anomaly_detection`，集成预测和评估功能。
- 打分时考虑多个因素：上下界、重构误差、历史/相对流量大小、异常持续时间和历史频率。
- **代码规范**: 在后续代码实现中，所有代码注释必须保留为英文 (Keep comments in English during implementation)。

**Non-Goals:**
- 替换现有的 `timeseries_forecast` 工具。
- 在本次迭代中实现复杂的监督机器学习模型进行打分（我们将从基于规则/策略的打分机制开始）。

## Decisions

- **新工具 vs 修改现有工具**: 我们将创建一个新工具 `anomaly_detection.py` (位于 `tool` 目录下)，其核心算法逻辑放置于 `analysis` 目录下，而不是修改 `timeseries_forecast.py`。这保持了预测和异常检测作为独立的关注点，允许 Agent 独立使用它们。
- **打分策略**: 分数（0 到 1）将是几个因素的加权总和或归一化组合：
  - **重构误差 (Reconstruction Error)**: 实际值与预测值之间的差异。
  - **边界违规 (Bound Violation)**: 实际值是否落在置信区间（如果可用）或计算的边界之外。
  - **持续时间 (Duration)**: 评估窗口中有多少个连续点是异常的。
  - **相对大小 (Relative Size)**: 与历史数据相比，指标的幅度。
- **评估窗口**: 算法将评估最近数据点的窗口以减少噪音，而不是仅评估单个点。

## Risks / Trade-offs

- **风险**: 基于规则的打分可能需要针对不同指标调整权重。
  - **缓解措施**: 暴露用于调整的参数，或使用适用于典型流量/业务指标的合理默认值。
- **风险**: 底层预测服务可能会失败或超时。
  - **缓解措施**: 优雅地处理预测错误，并向 Agent 返回信息丰富的错误消息。
